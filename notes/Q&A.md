### Q1：向量是一个点？求夹角不是应该要两条边吗？

**日期：** 2026-07-05

**问题：**
在 `Word-Embedding.py` 中，每个词（如 `king = [0.9, 0.8]`）看起来只是一个点，但余弦相似度要算两个向量的夹角，夹角不是需要两条边吗？

**解答：**

向量不是"点"，而是**从原点 O(0,0) 指向该点的有向线段（箭头）**。

| 概念 | 实际含义 |
|------|---------|
| 一个 `[0.9, 0.8]` 看起来是一个点 | 它同时也是**从原点指向该点**的箭头 |
| 求夹角的两条边从哪来？ | **原点→king** 是一条边，**原点→queen** 是另一条边 |
| 余弦相似度在算什么？ | 算这两个箭头**方向有多一致**（夹角越小越相似） |

两条边的公共起点就是**原点 O**，所以夹角自然存在。

可视化图片：`notes/images/word-embedding/vector_visualization.png`

- king 与 queen 的夹角 θ ≈ 29.1°，余弦相似度 ≈ 0.874
- `king - man + woman = [0.9, 0.2]` 与 queen 完全重合，夹角 = 0°，相似度 = 1.0

### Q2：大模型术语中，"梯度"是啥意思？

**日期：** 2026-07-05

**问题：**
训练大模型时总说"梯度下降""梯度爆炸""梯度消失"，梯度到底是什么？

**解答：**

梯度就是 **误差的"坡度"**——衡量模型参数往哪个方向调整、调多大，能让损失（loss）降得最快。

| 类比 | 说明 |
|------|------|
| 你在山上下山 | 模型训练就是找"最低点"（损失最小） |
| 脚下斜坡的方向 | **梯度方向**——往哪走坡度最陡 |
| 斜坡的陡峭程度 | **梯度大小**——步子该迈多大 |
| 逆着梯度走一步 | **梯度下降**——往坡底（最优参数）靠近一步 |
| 一步迈太大直接滚下山 | **梯度爆炸**——参数跳飞，loss 变成 NaN |
| 走到平地感觉不到坡 | **梯度消失**——参数几乎不动，模型学不动了 |

**那"损失函数"是什么？**

损失函数（Loss Function）是衡量模型预测结果与正确答案之间差距的"记分员"。

- 预测越准 → loss 越小（得分高）
- 预测越离谱 → loss 越大（得分低）
- 大模型常用 **交叉熵损失（Cross-Entropy Loss）**：预测下一个词时，给正确词概率打高分，其他词打低分，分数越偏 loss 越高

**梯度就是损失函数的"坡度"**——先算出 loss（错多少），再算梯度（往哪调），循环往复直到 loss 降到最低。

**数学上：** 梯度是损失函数对每个参数的偏导数的向量。

- 每个参数（权重 w）都有一个梯度值，正 → 减小该参数；负 → 增大该参数
- 参数更新公式：`w_new = w_old - 学习率 × 梯度`
- LLM 有数十亿参数，反向传播（backpropagation）一次性算出所有参数的梯度，然后统一更新

**一句话：** 梯度告诉模型"你现在错在哪、差了多少、该怎么调"。

### Q3：残差连接和归一化又是啥意思？

**日期：** 2026-07-05

**问题：**
Transformer 里反复出现"Add & Norm"（残差连接 + 层归一化），它们分别干什么用？

**解答：**

**一、残差连接（Residual Connection / Skip Connection）**

就是给网络搭一条"捷径"——让输入跳过一些层直接传到后面。

| 类比 | 说明 |
|------|------|
| 你在抄笔记 | 原句是"底稿"，经过几轮提炼可能走样 |
| 残差连接 = 原稿贴在旁边 | 输出 = **层处理结果 + 原始输入**，随时能对照原文 |
| 公式 | `output = Layer(input) + input` |

**为什么需要它？**

- **防退化：** 网络太深时，层数堆多了反而学不动（退化问题）。捷径让模型可以选择"这层没啥用，我直接跳过你"
- **防梯度消失：** 反向传播时梯度能沿着捷径直接流回浅层，不至于乘着乘着就没了
- Transformer 里每个 Attention 和 FFN 后面都跟一个残差连接

**二、层归一化（Layer Normalization）**

把数据的"尺子"统一——让每个样本的数值分布变得规整。

| 类比 | 说明 |
|------|------|
| 全班考试成绩 | 有人考 10 分，有人考 100 分，直接放在一起不好比较 |
| 归一化 = 统一换算成标准分 | 均值归 0，方差归 1，所有人放到同一把尺子上比 |

**为什么需要它？**

- **训练稳定：** 防止某些值太大或太小导致梯度爆炸/消失
- **加速收敛：** 所有层的数值范围一致，梯度步长更合理，学得更快
- Transformer 对每个 token 自己做归一化（LayerNorm），而不是跨 batch 做（BatchNorm）

**三、合起来看（Add & Norm）**

```
输入 → [Self-Attention] → + 原始输入（残差连接） → 归一化 → [FFN] → + 原始输入（残差连接） → 归一化 → 输出
```

1. **Add（残差）：** 把原始信息"捞回来"，防止丢失
2. **Norm（归一化）：** 把数值"拉平整"，防止跑偏

**一句话：** 残差连接是"抄近道保底"，归一化是"统一度量衡"——两个一起保证深层网络既能学得动，又能学得稳。

> **⚠️ 小贴士：别把 LayerNorm 和 Softmax 搞混**
>
> | 概念 | 操作 | 结果 |
> |------|------|------|
> | **Softmax** | 每个数 ÷ 总和 | 加起来 = 1，变成概率 |
> | **LayerNorm** | (每个数 - 均值) ÷ 标准差 | 均值 = 0，方差 = 1 |
>
> Softmax 是把分数"转成概率"，LayerNorm 是把数值"统一尺子"，两个都会出现在 Transformer 中。

### Q4：`if __name__ == '__main__':` 这行代码的作用是什么？

**日期：** 2026-07-05

**问题：**
每个 Python 文件里几乎都有这行，去掉好像也能跑，它到底在干嘛？

**解答：**

作用是：**判断当前文件是被直接运行，还是被别的文件导入（import）**。

**核心逻辑：**

| 场景 | `__name__` 的值 | 是否执行 `if` 里面的代码 |
|------|----------------|------------------------|
| 直接运行 `python 文件.py` | `'__main__'` | ✅ 执行 |
| 被其他文件 `import` | 模块名（如 `'Word-Embedding'`） | ❌ 跳过 |

**类比：**

> 你写了一本使用手册（.py 文件）。
> - 直接翻开来读 → 你是它的"主角"（`__main__`），从头读到尾
> - 别人拿它当参考书引用 → 只需要查目录和定义，不想看到里面的示例和自测题

**所以 `if __name__ == '__main__':` 就是在说：**

> "当我是主角时我才跑这段代码；别人把我当工具引用时，只提供函数和类，不执行测试代码。"

**实际写法：**

```python
def train():
    print("训练模型...")

# 下面这个 if 块只在本文件直接运行时执行
if __name__ == '__main__':
    train()          # 写测试代码、示例、临时调试
```

**一句话：** 让一个 `.py` 文件既可以当"可执行脚本"直接跑，又可以当"模块"被别的文件导入调用，互不干扰。

### Q5：类里面的 `__init__` 是干什么的？

**日期：** 2026-07-05

**问题：**
写类的时候总要写一个 `def __init__(self):`，它有什么用？

**解答：**

`__init__` 是类的**构造方法**——也叫"初始化方法"，当你创建一个对象时自动调用，用来给新对象设置初始值。

**类比：**

> 类 = 一张蛋糕模具（蓝图）
> `__init__` = 往模具里倒面糊 + 设定烘烤参数
> 对象 = 烤出来的那块蛋糕

**直观对比：**

```python
class Student:
    # __init__：给每个新学生设定名字和年龄
    def __init__(self, name, age):
        self.name = name   # 记录名字
        self.age = age     # 记录年龄

# 创建对象时自动触发 __init__
s1 = Student("张三", 20)   # 自动调用 __init__，self.name="张三", self.age=20
s2 = Student("李四", 22)

print(s1.name)  # 张三
print(s2.name)  # 李四
```

| 关键点 | 说明 |
|--------|------|
| `self` | 指代"当前正在被创建的那个对象自己" |
| `__init__` 是自动调用的 | 你 `Student("张三", 20)` 一写，它就自动执行 |
| 不写 `__init__` 行不行 | 行，但每个对象创建后需要手动设属性，麻烦 |

**一句话：** `__init__` 是"对象出生时的初始化仪式"——给每个新对象填好它该有的数据。

### Q6：可以把 `Dict` 理解为对象吗？

**日期：** 2026-07-05

**问题：**
字典（Dict）存的是键值对，感觉有点像对象有属性名和属性值，它算对象吗？

**解答：**

**严格来说：Dict 不是对象，但两者确实很像。**

| 对比维度 | 字典（Dict） | 对象（Object / 实例） |
|----------|-------------|----------------------|
| 本质 | 一个**数据结构**（存数据的容器） | 一个**类的实例**（既有数据又有行为） |
| 存取方式 | `dict["key"]` | `obj.属性名` |
| 能附加方法吗 | ❌ 不能 | ✅ 可以定义自己的函数 |
| 灵活度 | 键可以随时增删改，极灵活 | 属性由类定义，相对固定 |

**为什么它们感觉很相似？**

Python 里的对象内部其实就是用 Dict 来存属性的：

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("小明", 18)
print(p.__dict__)  # {'name': '小明', 'age': 18}  ← 本质就是一个字典！
```

所以字典可以看作对象的"骨架"——对象 = 字典 + 方法 + 类型约束。

**实际怎么选？**

| 场景 | 用 Dict | 用类/对象 |
|------|---------|-----------|
| 只是暂存几个字段，没有行为 | `{"name": "张三", "age": 20}` | 简单轻量 |
| 需要反复使用、有方法逻辑 | | 用类组织更清晰 |
| 数据格式不确定，动态增删字段 | `data["extra"] = xxx` | 字段随时变，dict 更方便 |

**一句话：** Dict 可以看作"简陋版对象"，对象可以看作"自带行为的字典"——字典管存数据，对象管数据和行为的封装。

### Q7：为什么有时候函数有 `self`，有时候没有？

**日期：** 2026-07-05

**问题：**
有的函数定义在类里面带 `self`，有的又不带，不带 `self` 的函数也不是不能写，它们有什么区别？

**解答：**

**`self` 代表"这个对象自己"**——只有需要访问或修改对象的数据时，才需要 `self`。

Python 中函数分三种情况：

| 类型 | 写法 | 第一个参数 | 能访问对象属性吗 | 什么时候用 |
|------|------|-----------|----------------|-----------|
| **实例方法** | `def 方法名(self):` | `self` | ✅ 能 | 需要读写对象的属性/数据 |
| **静态方法** | `@staticmethod` `def 方法名():` | 无特殊参数 | ❌ 不能 | 跟类有关但不需要对象数据（工具函数） |
| **普通函数** | `def 函数名():` | 无特殊参数 | ❌ 不能 | 写在类外面，跟类无关 |

**直观对比：**

```python
class Student:
    school = "一中"  # 类属性

    def __init__(self, name):
        self.name = name  # 实例属性

    # 1️⃣ 需要 self —— 因为要访问每个学生自己的名字
    def introduce(self):
        print(f"我叫{self.name}")  # self.name 每个对象不同

    # 2️⃣ 不需要 self —— 跟具体学生无关，只是跟"学生"这个概念有关的工具
    @staticmethod
    def get_school():
        return Student.school  # 只用了类属性，没用 self

    # 3️⃣ 不需要 self —— 只是功能上归类组织，不碰任何数据
    @staticmethod
    def info():
        print("这是一个学生类")

# ---- 调用 ----
s = Student("小明")
s.introduce()          # ✅ 实例方法，自动传 self

Student.get_school()   # ✅ 静态方法，不用创建对象也能调用
Student.info()         # ✅ 静态方法
```

**简单记忆：**

> - **需要 self** → 你要用 `self.name`、`self.age` 这类"每个对象独有的数据"
> - **不需要 self** → 你只是干一件跟类/对象有关但不依赖具体对象的事

**扩展：还有一个 `@classmethod` 用 `cls` 而不是 `self`：**

```python
class Student:
    count = 0
    
    @classmethod
    def show_count(cls):
        print(f"当前学生总数: {cls.count}")  # 访问的是类变量，不是某个对象的变量
```

| 参数 | 用在 | 指向 |
|------|------|------|
| `self` | 实例方法 | 当前对象 |
| `cls` | 类方法 `@classmethod` | 当前类本身 |
| 无特殊参数 | 静态方法 `@staticmethod` / 普通函数 | 啥也不指向 |

**一句话：** 要不要 `self`，取决于你需不需要碰"每个对象专属的数据"——需要就写，不需要就不写。
