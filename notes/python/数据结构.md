## Q1：可以把 `Dict` 理解为对象吗？

**日期：** 2026-07-05

**严格来说：Dict 不是对象，但两者确实很像。**

| 对比维度 | 字典（Dict） | 对象（Object / 实例） |
|---|---|---|
| 本质 | 一个**数据结构**（存数据的容器） | 一个**类的实例**（既有数据又有行为） |
| 存取方式 | `dict["key"]` | `obj.属性名` |
| 能附加方法吗 | ❌ 不能 | ✅ 可以定义自己的函数 |
| 灵活度 | 键可以随时增删改，极灵活 | 属性由类定义，相对固定 |

### 为什么它们感觉很相似？

Python 里的对象内部其实就是用 Dict 来存属性的：

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("小明", 18)
print(p.__dict__)  # {'name': '小明', 'age': 18}  ← 本质就是一个字典！
```

所以字典可以看作对象的"骨架"——对象 = 字典 + 方法 + 类型约束。

### 实际怎么选？

| 场景 | 用 Dict | 用类/对象 |
|---|---|---|
| 只是暂存几个字段，没有行为 | `{"name": "张三", "age": 20}` 简单轻量 | |
| 需要反复使用、有方法逻辑 | | 用类组织更清晰 |
| 数据格式不确定，动态增删字段 | `data["extra"] = xxx` 字段随时变，dict 更方便 | |

### 一句话

Dict 可以看作"简陋版对象"，对象可以看作"自带行为的字典"——字典管存数据，对象管数据和行为的封装。
